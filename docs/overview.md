# 背景

- 该项目是一个多人在线卡牌小游戏
- 使用 web 相关技术栈，目前展示不需要考虑美术相关资源，主要实现逻辑相关部分
- 前端采用 react
- 后端使用 python

# 游戏规则

## 游戏中卡牌

班长

- 调和值：2
- 描述：选定一个玩家，两个人各自选定一张手牌进行交换。
- 胜利条件：4 调和成功即可获胜

图书委员

- 调和值：1
- 描述：你可以查看放在调和区的所有卡牌。
- 胜利条件：4 调和成功即可获胜

外星人

- 调和值：-1
- 描述：当你持有这张牌时，若有人使用了优等生的特技，你可以假装是犯人来回应他。该牌正面打出的时候没有任何特殊效果。
- 胜利条件：1 被监禁即可获胜

归宅部

- 调和值：0
- 描述：选择一张自己的手牌，和调和区的一个卡牌进行替换。
- 胜利条件：5 没有任何人获胜即可获胜

保健委员

- 调和值：1
- 描述：选择一张已经被正面打出的卡牌，将它归入自己的手牌。
- 胜利条件：4 调和成功即可获胜

风纪委员

- 调和值：1
- 描述：你可以查看一名玩家的所有手牌。
- 胜利条件：4 调和成功即可获胜

新闻部

- 调和值：1
- 描述：每个玩家选择自己的一张手牌，递给下一个玩家。
- 胜利条件：4 调和成功即可获胜

大小姐

- 调和值：1
- 描述：选择一个玩家，将他的手牌中的 1 张卡牌，放到自己的手牌中，然后选择一张自己的手牌给该玩家（可以是刚从该玩家收到的牌）。
- 胜利条件：4 调和成功即可获胜

共犯

- 调和值：0
- 描述：将一张放置再玩家面前的质疑位置的牌转移到任意一个其他玩家的质疑位置。（不可以是自己）
- 胜利条件：3 犯人获胜即可获胜

感染者

- 调和值：0
- 描述：在下一个回合开始时，若这张牌仍然正面朝上摆在你面前，则可以将调和区的一张牌加到自己手牌中。
- 胜利条件：2 调和失败即可获胜

犯人

- 调和值：0
- 描述：此卡片无法被正面朝上或反面朝上打出，但可以被其他卡片的效果移动。
- 胜利条件：3 不被监禁即可获胜

犯人

- 调和值：0
- 描述：此卡片无法被正面朝上或反面朝上打出，但可以被其他卡片的效果移动。
- 胜利条件：3 不被监禁即可获胜

学生会长

- 调和值：3
- 描述：正面打出无效果。
- 胜利条件：4 调和成功即可获胜

优等生

- 调和值：2
- 描述：正面打出该卡时，其他玩家闭上眼睛，同时持有犯人卡的玩家需要举手示意。
- 胜利条件：3 不被监禁即可获胜

## 游戏人数确定

确认游戏人数，手牌分配规则：

- 3–4 人时：每人分到 6 张 手牌
- 5 人时：每人分到 5 张 手牌

## 出卡规则

从手牌中选出 1 张牌，并从以下的三种使用方式中选择一种来执行：

A. 特技：将卡牌正面朝上放置在自己面前，并执行能力栏中的效果。
B. 调和：将卡牌背面朝上放置在少女的尸体旁边，用以调和防腐剂。
C. 质疑：将卡牌背面朝上放置在其他玩家面前。质疑该玩家为凶手。玩家不能质疑自己。

先手规则：

- 分配到学生会长的玩家首先出卡，按照顺时针顺序进行。

等待结算阶段：

- 当某玩家手牌只剩一张时，该玩家视为处于**等待结算阶段**：本回合不能出牌，回合会自动跳过；且**不能被需要移动/交换牌的特技选中**（如班长、大小姐、保健委员等均不能以该玩家为目标）。**风纪委员仅查看**，可以以该玩家为目标查看其手牌。质疑仍可以以该玩家为目标。所有人手牌都剩一张时，进入胜败判定阶段。

## 胜利条件

当所有人都只剩一张手牌时，便进入胜败判定阶段。

- 判定 1：调和值
  - 查看是否达到游戏人数对应的调和值。
- 判定 2：质疑结算
  - 结算质疑牌上的数值总和，数值最大的人将被监禁。(如果有多个人数值相同且最大（>0），则这些玩家都认为是被监禁)
- 判定 3：胜利条件判定
  - 玩家按照胜利条件低优先级依次公开自己的手牌，并判定是否满足胜利条件。

胜利优先级表，可能存在多个玩家满足胜利条件，此时按照胜利优先级，确定哪一个玩家胜利。


| 角色名  | 胜利条件   | 胜利优先级 |
| ---- | ------ | ----- |
| 外星人  | 被监禁    | 1     |
| 感染者  | 调和失败   | 2     |
| 犯人   | 不被监禁   | 3     |
| 共犯   | 犯人胜利   | 3     |
| 学生会长 | 调和成功   | 4     |
| 班长   | 调和成功   | 4     |
| 保健委员 | 调和成功   | 4     |
| 大小姐  | 调和成功   | 4     |
| 图书委员 | 调和成功   | 4     |
| 风纪委员 | 调和成功   | 4     |
| 新闻部  | 调和成功   | 4     |
| 归宅部  | 无任何人胜利 | 5     |


# 技术选型

## 前端

- 使用 react 作为前端框架
- 尽可能简单，不要过于复杂
- 暂时不需要比较复杂的特效，先专注于游戏细节的实现

## 后端

- 使用 python 进行后端的构建。
- 游戏本身是一个多人在线卡牌游戏，支持这个。
- 实现尽量简洁。
- 暂时不需要支持多个房间，只需要有一个房间即可。
